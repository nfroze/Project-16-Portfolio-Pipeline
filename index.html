Selected Projects

Five projects that demonstrate core DevSecOps capabilities — from security-integrated pipelines to multi-cloud orchestration.

01

DevSecOps Pipeline

The Challenge: Security scanning typically happens too late — after deployment, after the vulnerability is already in production.

The Solution: A CI/CD pipeline with security gates at every stage. Code commits trigger SAST scanning with Semgrep and secrets detection with Gitleaks. Container builds run through Trivy for vulnerability scanning. Infrastructure changes pass through Checkov before Terraform applies. GuardDuty monitors runtime, feeding alerts to Splunk SIEM.

The Outcome: Vulnerabilities caught at commit, not in production.

Amazon EKS Terraform GitHub Actions Checkov Trivy Semgrep GuardDuty Splunk

02

Multi-Cloud Orchestration

The Challenge: Vendor lock-in limits flexibility and negotiating power. Most teams know one cloud well and struggle to move workloads.

The Solution: Terraform modules that abstract cloud-specific resources behind a consistent interface. One workflow deploys equivalent infrastructure to AWS, Azure, or GCP. Ansible handles configuration management. Datadog provides unified observability across all three clouds.

The Outcome: True portability — same code, any cloud.

Terraform Ansible AWS Azure GCP Datadog GitHub Actions
View on GitHub
03

MCP Security Operations

The Challenge: When GuardDuty detects a threat, the clock starts. Manual response means minutes or hours before containment — plenty of time for lateral movement.

The Solution: A Model Context Protocol server that gives AI assistants direct access to security operations. GuardDuty findings trigger EventBridge rules. Lambda functions isolate compromised EC2 instances automatically. The team gets Slack notifications with full context.

The Outcome: Threat detected → server isolated → team notified in seconds, not hours.

MCP AWS GuardDuty EventBridge Lambda Slack API Node.js

04

AI Developer Platform

The Challenge: Developers waste time navigating scattered documentation, provisioning environments, and tracking ML experiments across disconnected tools.

The Solution: An internal developer platform built on Backstage. Service catalog centralises documentation and ownership. ArgoCD handles GitOps deployments through Helm charts. MLflow tracks experiments and model versions. Hybrid EKS/ECS architecture balances cost and performance.

The Outcome: One platform for services, deployments, and ML — reducing onboarding friction.

Backstage Amazon EKS Amazon ECS MLflow ArgoCD Helm Terraform
View on GitHub
05

AI Governance Framework

The Challenge: ML teams deploy models without guardrails. Unapproved models, missing documentation, compliance violations — all discovered after the fact.

The Solution: Policy-as-code enforcement at two levels. HCP Terraform Sentinel policies validate infrastructure before provisioning. OPA Gatekeeper admission controllers enforce model requirements at Kubernetes deployment time. Nothing deploys without passing governance checks.

The Outcome: Compliance by default — unapproved models blocked before they reach production.

HCP Terraform Sentinel OPA Gatekeeper Amazon EKS Kubernetes
